# Faraday — repo conventions

## Git workflow (required)
- **Split commits**: one coherent, self-contained change per commit. Don't lump unrelated changes together.
- **Short messages**: concise subject line, no long bodies. No bullet-point essays.
- **No Claude signature**: do NOT add `Co-Authored-By: Claude ...` or any "Generated with Claude" trailer.
- **Never push**: stage and commit only. The user always pushes.
